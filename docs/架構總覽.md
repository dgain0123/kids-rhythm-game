# 架構總覽

幼兒節奏遊戲：小朋友對麥克風**真的打鼓**，畫面同時顯示**鼓譜**與**可愛角色**，判定過關/失敗。純靜態網站（HTML+CSS+JS），無後端。

## 檔案結構
```
index.html            遊戲畫面 + PWA 標頭 + 註冊 SW
css/style.css         幼兒風格樣式
js/
  main.js             總串接：目錄/關卡/麥克風/音效/狀態/SW 註冊
  audio.js            麥克風 + 打鼓偵測(DrumListener)
  game.js             關卡判定(打夠過關/超過失敗)
  render.js           畫譜面音符(四分/八分/符樑) + 過關彩帶
  characters.js       角色系統(多隻並排 + 號碼牌)
charts/level1..8.json 各關的譜(唯一資料來源)
characters/
  index.json          角色清單(圖片或 emoji)
  cat.png rabbit.png  去背角色圖
sounds/
  cheer.wav/.mp3      過關真人歡呼
  fail.wav            失敗音效
manifest.webmanifest  PWA 設定(可加到主畫面)
sw.js                 Service Worker(離線快取)
icon-192/512.png      App 圖示(小鼓)
serve.py              本機伺服器(禁快取 + 多執行緒)
關卡/第N關.md          每關的規格正解檔
docs/*.md             架構/流程文件(本資料夾)
```

## 一場遊戲的資料流
1. 開啟 → `main.js` 註冊 SW、`initCharacters` 建角色、`buildLevelMenu` 讀各關標題建目錄 → 顯示**關卡目錄**（或 `?level=N` 直接進關）。
2. 選一關 → `goToLevel` 載入 `charts/levelN.json`，`drawNotes` 畫譜面，`setCharCount(maxHits)` 決定顯示幾隻角色。
3. 按「開始遊戲」(使用者手勢) → `unlockAudio()`+`primeCheer()` 解鎖音訊 → `DrumListener.start()` 開麥克風 → `new Game()` 開始判定。
4. 打鼓 → `DrumListener` 偵測到一下 → `game.registerHit()` → 狀態機 `progress/hitOnce/pass/fail` → `onState()` 更新畫面。
5. 過關 → 播歡呼(`celebrateSound`)+彩帶+拉炮；失敗 → 播 `fail`；都出現「再玩1次」，有下一關則出現「下一關 →」。

## 關鍵技術決策(細節見各專題文件)
- 麥克風需安全環境(localhost/https)，故本機用 `serve.py`、對外用 https。見 [打鼓偵測](打鼓偵測.md)。
- Safari 音訊限制多：音效用「點擊時解鎖的 `<audio>` 元素 + blob 記憶體載入」。見 [音效系統](音效系統.md)。
- 角色數＝該關要打幾下；每隻頭上號碼牌。見 [角色與號碼牌](角色與號碼牌.md)。
- 關卡資料驅動(charts/*.json)，目錄自動生成。見 [關卡系統](關卡系統.md)。
- PWA 離線：Service Worker「網路優先/離線用快取」+ 大檔範圍請求處理。見 [離線與部署](離線與部署.md)。
- 一路踩過的坑集中在 [踩過的坑](踩過的坑.md)。
