# 音效系統（js/main.js）

過關真人歡呼、失敗音效。這塊在 Safari 上折騰最久，最終方案如下。

## 檔案
- 過關：`sounds/cheer.wav`（優先）/ `cheer.mp3`（真人歡呼，使用者提供）
- 失敗：`sounds/fail.wav`（使用者提供）
- 若都沒有 → 用合成人聲歡呼(ya/yeah/woohoo 共振峰)當後備

## 播放管線（為什麼這樣做）
這台 Safari 上：**能穩定出聲的是 HTML `<audio>` 元素**，Web Audio 合成/buffer 在此機常無聲。所以真人音效一律用 `<audio>` 元素播。

### 兩個 Safari 關卡
1. **自動播放限制**：Safari 只允許「使用者點擊」觸發的音訊。過關是「麥克風偵測」觸發（非點擊）會被擋。
   - 解法：在按「開始遊戲」的點擊裡呼叫 `primeCheer()` — 把 `<audio>` **靜音播一下馬上暫停**（primeEl），元素就取得播放許可，之後從任何情境都能播。
   - `primeCheer` 每次開始遊戲都試著解鎖（音檔可能還在載，載好後的下一次點擊才解鎖得到）。
2. **大檔離線播不出**：`<audio>` 播大檔(歡呼 856KB)會發「範圍請求」，離線時 SW 只回完整檔→Safari 拒絕→無聲（失敗音小所以沒事）。
   - 解法：預載時把音檔**整包 `fetch`→`blob`→`URL.createObjectURL`** 建 `<audio>`（`loadAudioEl`）。播放時從記憶體播、**完全不發網路/範圍請求** → 大檔離線也直接有聲，跟失敗音一樣。

## 流程
- `preloadCheer()`（開啟時）：用 `loadAudioEl` 把 fail 與 cheer 整包載進記憶體成 `failEl` / `cheerEl`。
- `unlockAudio()`（開始遊戲點擊）：播一個極短靜音 buffer 解鎖 Web Audio（給合成後備用）。
- `primeCheer()`（開始遊戲點擊）：解鎖 `cheerEl`、`failEl`。
- 過關 `celebrateSound()`：優先播 `cheerEl`；沒有才退 Web Audio buffer→合成人聲。
- 失敗 `failSound()`：播 `failEl`。
- 「聲音先播、彩帶延後 120ms」：避免彩帶第一幀卡主執行緒壓到聲音。

## 試聽
畫面有「🔊 試聽歡呼」按鈕：單獨播歡呼（不玩遊戲、不放彩帶、不碰麥克風），用來隔離「斷掉/沒聲」是檔案還是遊戲負載。

## 換音效
- 換歡呼：覆蓋 `sounds/cheer.wav`（或 mp3）。
- 換失敗音：覆蓋 `sounds/fail.wav`。`sounds/fail_samples/` 有 10 個合成樣本 + 試聽頁 `preview.html`。
- 換完記得同步到 -public 並更新 SW 快取（見 [離線與部署](離線與部署.md)）。
