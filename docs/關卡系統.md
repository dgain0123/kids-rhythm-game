# 關卡系統

關卡是**資料驅動**的：一個 `charts/levelN.json` = 一關，目錄與畫面自動生成。

## 關卡清單
`js/main.js` 的 `LEVELS = ["level1", ..., "level8"]` 決定順序。加關就加一個 json + 一行。

## 譜格式 charts/levelN.json
```json
{
  "title": "第2關：打2下",
  "maxHits": 2,
  "hint": "看到兩個音符，打2下鼓！",
  "notes": [
    { "type": "quarter", "drum": "snare" },
    { "type": "quarter", "drum": "snare" }
  ]
}
```
- `maxHits`：要打幾下（＝角色數＝過關所需）。
- `notes[].type`：`quarter`(四分) / `eighth`(八分)。`drum`：鼓件。

## 目前 8 關
| 關 | maxHits | 譜面 |
|---|---|---|
| 1~4 | 1~4 | 對應數量的四分音符 |
| 5 | 5 | 2 八分(1組) + 3 四分 |
| 6 | 6 | 4 八分(2組) + 2 四分 |
| 7 | 7 | 6 八分(3組) + 1 四分 |
| 8 | 8 | 8 八分(4組) |

## 譜面繪製（js/render.js `drawNotes`）
- 畫五線譜 + 音符（黑色實心符頭 + 符桿）。小鼓＝五線譜第三間。
- **依「拍」定位**：四分=1拍、八分=半拍，每拍等寬（兩個八分＝一拍＝一組）。
- 八分音符：**每兩個一組(一拍)打符樑**（`drawBeam`），組跟組分開；落單的畫旗子（`drawFlag`）。

## 判定（js/game.js）
狀態機：`ready → progress → hitOnce → pass/fail`。
```
每打一下 hits++：
  hits > maxHits           → fail（打太多）
  hits < maxHits           → progress（還沒打夠，繼續等，不失敗）
  hits == maxHits          → hitOnce，開 1.5 秒視窗；沒再多打 → pass
```
- **不判拍子**：八分/四分只是譜面外觀，實際每個音符＝打一下。
- 「打夠才過關、超過才失敗、不夠繼續等」對幼兒友善（零挫折）。

## 目錄與導覽（js/main.js）
- 一進來顯示**關卡目錄**（`buildLevelMenu` 讀各關 json 標題建卡片）。
- 點卡片 → `selectLevel` → `goToLevel` 載譜、畫面、角色數 → 顯示「開始遊戲」。
- 遊戲中「☰ 選關」回目錄；過關後「下一關 →」進下一關；`?level=N` 直接進某關。

## 不劇透的設計（使用者要求）
- 打鼓過程**不顯示**「還剩幾下」等文字（progress/hitOnce 狀態列留空）。
- 打到剛好那下角色**不放大不歪斜**（拿掉了會露餡的提示）。
- 不閃黃回饋（拿掉了）。小朋友純看音符/角色數量自己數。

## 每關的規格正解檔
`關卡/第N關.md` 是各關規則的唯一正解檔；改某關先改它，程式/資料以它為準。每個 md 維持 200 行內。
