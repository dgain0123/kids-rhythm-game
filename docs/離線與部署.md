# 離線與部署

## 本機伺服器 serve.py
- 麥克風需 `localhost`/https，故本機用 `serve.py`（不能直接雙擊 index.html）。
- **禁快取**：回 `Cache-Control: no-store...`，改檔普通重整就生效（終結舊快取問題）。
- **多執行緒**：`ThreadingHTTPServer`，避免瀏覽器同時多請求把單執行緒卡死。
- 啟動：`python3 serve.py` 或雙擊 `start.command`。

## PWA（可離線、可加到主畫面）
三個檔案：
- `manifest.webmanifest`：App 名稱、圖示、`display: standalone`、`start_url/scope: "./"`（相對路徑，本機與 Pages 子路徑都適用）。
- `icon-192.png` / `icon-512.png`：小鼓圖示。
- `sw.js`：Service Worker，負責離線快取。
- `index.html` 有 manifest/theme-color/apple-touch-icon 標頭；`main.js` 註冊 SW。

### Service Worker 策略（sw.js）
- **網路優先，離線用快取**：有網路→抓最新並更新快取（線上永遠最新）；沒網路→用快取。
- 安裝時**預抓核心檔案**(CORE)：8 關譜、音效、程式、角色圖、圖示等。用 `Promise.allSettled` 個別加入，缺檔不會整包失敗。
- 快取鍵**去掉查詢字串**(`?t=...`)，避免無限累積。
- 只快取完整檔(status 200)，不快取 206 片段。
- **範圍請求處理**：離線時大音檔(歡呼)的 Range 請求 → 從完整快取切片段回 206（否則 Safari 播不出）。
  - 註：音效已改 blob 記憶體載入(見 [音效系統](音效系統.md))，播放時不發範圍請求；此處理為額外保險。

### 加到主畫面
- 手機 Safari：分享 → 加入主畫面；Android Chrome：⋮ → 安裝應用程式。
- Mac：Safari(Sonoma+) 檔案→加入 Dock；舊 macOS 用 Chrome/Edge 安裝，或只加書籤(離線一樣有效)。

## GitHub Pages 部署
- 公開版 repo：`dgain0123/kids-rhythm-game`（gh CLI 已登入 dgain0123）。免費 Pages 需 **repo 公開**。
- 部署＝ `git push` 後 GitHub 自動建置。網址 `https://dgain0123.github.io/kids-rhythm-game/`。
- **CDN 延遲**：push 後 Pages 建置 + CDN 傳播要**數分鐘**，期間可能還給舊檔。驗證方法：
  ```
  curl -s https://dgain0123.github.io/kids-rhythm-game/js/main.js | grep -c 關鍵字
  gh api repos/dgain0123/kids-rhythm-game/pages/builds/latest   # status 要 built
  ```
  確認 build=built 且線上檔含新內容，再叫使用者重整（否則會誤以為沒生效）。

## 本機↔公開版同步流程
1. 本機改完、測好。
2. `rsync -a js/ css/`、`cp index.html sw.js 音檔 圖 charts/...` 到 `幼兒節奏遊戲-public/`。
   - **公開版角色圖**：貓/兔已確認無版權，與本機一致。
3. 兩邊各自 `git commit`；-public `git push`。
4. 等 Pages 建置 + CDN，驗證線上是新版，再請使用者連網重整（SW 更新常要載入一次才切換，重整 2 次較保險）。
