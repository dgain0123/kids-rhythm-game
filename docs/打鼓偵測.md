# 打鼓偵測（js/audio.js）

用麥克風偵測「真的打鼓」的那一下。核心是 `DrumListener`。

## 原理
1. `getUserMedia` 取得麥克風 → `AudioContext` + `AnalyserNode`。
2. 每一幀取時域資料，算**峰值(peak)**＝最大絕對振幅（比 RMS 更抓得到打擊的瞬間爆音）。
3. **上升邊緣**偵測：峰值衝過「有效門檻」且距上次夠久、且已武裝 → 算一下（`onHit`）。

## 有效門檻（自適應）
```
trigger = max(使用者設的絕對門檻, 背景底噪 × riseFactor + margin)
```
- 房間吵時門檻自動抬高、安靜時維持設定 → 兩邊都穩。
- 底噪 `_ambient` 用 EMA 緩慢跟隨（打擊當下不更新，避免把打擊算進底噪）。

## 防誤觸的幾個機制
- **暖機 `warmupMs=600`**：麥克風剛開/恢復的前 0.6 秒不觸發（macOS 開麥克風有初始化爆音，否則被算成一下）。
- **不反應期 `refractoryMs=130`**：打一下後這段時間內不再觸發。**不能太長**，否則快速打的第二下被吃掉（曾設 400ms 出過這問題）。
- **遲滯武裝**：要等峰值掉回接近底噪才「重新武裝」，靠這個防餘音/回音重複觸發（而不是靠長不反應期）。

## 靈敏度滑桿
- HTML 滑桿值 0~1（左=不靈敏、右=很靈敏），預設 **0**（最不靈敏，適合真鼓較大聲）。
- `sensToThreshold(s) = THRESH_MAX - s*(THRESH_MAX-THRESH_MIN)`，往「很靈敏」拉→門檻變低。
- ⚠️ 曾把滑桿方向做反(往很靈敏反而門檻最高)，已修正。

## 生命週期（重要）
- `start()`：開麥克風、開始迴圈。
- `pause()`：**只暫停偵測、保持麥克風開著**。過關/失敗時用這個，**不要 close**——關閉會讓 macOS 重設音訊裝置，害輸出閃斷(歡呼被切)。
- `resume()`：恢復偵測，**並喚醒被瀏覽器 suspend 的 AudioContext**（否則換關量表全 0）。
- `stop()`：完全關閉釋放麥克風（遊戲進行中不用）。
- `isLive()`：串流是否還活著；`startGame` 發現死掉就丟掉重開。

## 重連按鈕
畫面有「🎤 重連麥克風」：完全 `stop()`+重建+重開本關，麥克風卡住時的救命鍵。

## 環境限制
- 麥克風需**安全環境**：`localhost` 或 **https** 才能開。故本機走 `serve.py`(localhost)、對外走 https(GitHub Pages)。
- **Mac mini 沒內建麥克風**，要接外接麥克風才能玩（見 [踩過的坑](踩過的坑.md)）。

## 診斷面板
網址加 `?debug=1` 會顯示即時音量、AudioContext 狀態、麥克風裝置名稱與錯誤，用來排查沒訊號的問題。
